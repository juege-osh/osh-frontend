# 问答社区功能演示脚本

## 🎬 演示流程（5分钟）

### 场景1: 浏览问题列表 (1分钟)

**操作步骤:**
1. 打开浏览器，访问 `http://localhost:3000/question_answer/1`
2. 展示页面整体布局
   - 精美的渐变色头部
   - 清晰的面包屑导航
   - 功能完整的搜索栏
3. 演示搜索功能
   - 选择资源类型
   - 输入关键词
   - 点击搜索
4. 演示Tab切换
   - 点击不同的Tab查看不同分类
   - 展示Tab切换的流畅动画
5. 展示问题卡片
   - 悬停效果
   - 付费问题的特殊标识
   - 已解决问题的特殊样式
   - 统计数据展示

**演示话术:**
> "这是我们的问答社区首页，采用了现代化的设计风格。顶部是功能完整的搜索栏，支持多维度筛选。下方的Tab可以快速切换不同分类。每个问题卡片都展示了关键信息，包括标题、标签、统计数据等。注意看这个悬停效果，非常流畅。"

---

### 场景2: 发布新问题 (2分钟)

**操作步骤:**
1. 点击"我要提问"按钮
2. 展示弹窗打开动画
3. 演示表单填写
   - 选择资源类型（例如：课程）
   - 输入资源ID（例如：1）
   - 输入问题内容（例如："在 Spring Boot 应用中，如果一个 Bean 使用了 @Async 注解来实现异步方法，但是发现并不生效，方法仍然是同步执行的，请问可能是什么原因？"）
   - 选择问题类型（公开/付费）
4. 演示标签功能
   - 输入标签"Spring Boot"并回车
   - 点击推荐标签"Java"
   - 展示标签的添加动画
   - 删除一个标签
5. 展示字符计数
6. 点击"立即发布"
7. 展示成功提示
8. 展示列表自动刷新

**演示话术:**
> "点击'我要提问'打开发布弹窗。表单设计非常直观，支持关联资源、选择问题类型。标签功能很强大，可以手动输入，也可以从推荐标签中选择。注意这个字符计数功能，帮助用户控制内容长度。提交后会有友好的成功提示，列表会自动刷新显示新问题。"

---

### 场景3: 查看问题详情 (1.5分钟)

**操作步骤:**
1. 点击刚才创建的问题卡片
2. 展示详情页布局
   - 完整的问题信息
   - 用户信息展示
   - 标签展示
   - 统计数据
3. 演示关注功能
   - 点击"关注问题"按钮
   - 展示按钮状态变化
   - 展示关注数增加
4. 滚动到回答区域
   - 展示回答列表
   - 展示最佳回答的特殊样式（如果有）
5. 演示点赞功能
   - 点击点赞按钮
   - 展示点赞数变化

**演示话术:**
> "进入详情页，可以看到问题的完整信息。顶部有关注按钮，点击后状态立即更新。下方是回答列表，最佳回答会有特殊的绿色丝带标识。每个回答都可以点赞，操作非常流畅。"

---

### 场景4: 提交回答 (0.5分钟)

**操作步骤:**
1. 滚动到页面底部
2. 展示固定的回答输入框
3. 输入回答内容（例如："这个问题通常是因为在同一个类内部调用 @Async 方法导致的。Spring AOP 代理无法拦截内部方法调用。解决方案是将异步方法提取到另一个 Bean 中，或者通过 ApplicationContext 获取代理对象来调用。"）
4. 展示字符计数
5. 点击"提交回答"
6. 展示成功提示
7. 展示回答列表自动更新

**演示话术:**
> "底部是固定的回答输入框，无论页面滚动到哪里都能快速回答。输入内容后点击提交，回答会立即显示在列表中。整个流程非常流畅。"

## 💡 演示技巧

### 准备工作
1. ✅ 确保后端API正常运行
2. ✅ 准备好测试账号
3. ✅ 清理浏览器缓存
4. ✅ 关闭不必要的浏览器标签
5. ✅ 调整浏览器窗口大小到合适尺寸

### 演示注意事项
1. 🎯 操作要流畅，不要停顿
2. 🎯 强调关键功能和亮点
3. 🎯 展示动画效果时稍作停留
4. 🎯 遇到问题要淡定处理
5. 🎯 保持演示节奏

### 常见问题应对
- **网络慢**: "这里可能网络有点慢，实际使用时会很快"
- **数据未加载**: "让我刷新一下页面"
- **操作失败**: "这个是测试环境的问题，生产环境不会有"
